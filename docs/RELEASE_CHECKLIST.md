# Release checklist (CRM Nemnovo)

## Перед релизом

- [ ] **Секреты/ENV**
  - [ ] `SECRET_KEY` задан и не дефолтный
  - [ ] `ALLOWED_HOSTS` не `["*"]` (перечень доменов)
  - [ ] `LOG_FORMAT=json`, `LOG_LEVEL=INFO`
  - [ ] Настроены секреты интеграций при необходимости:
    - [ ] `SITE_WEBHOOK_SECRET`
    - [ ] `TELEPHONY_WEBHOOK_SECRET`
    - [ ] `SMS_API_KEY`, `SMS_SENDER`
    - [ ] `YOOKASSA_*` (если включаем оплату)

- [ ] **База/миграции**
  - [ ] `alembic upgrade head` выполнен
  - [ ] Если БД была “старой” (без миграций) — применены ручные изменения из `docs/RUN.md`
    - [ ] `companies.segment`
    - [ ] `invoices.issuer_company_id`
    - [ ] `stock_movements` (таблица)

- [ ] **Пользователи/роли**
  - [ ] `python scripts/seed.py` выполнен (или `./scripts/run_seed.sh`)
  - [ ] Есть минимум 1 администратор, который может войти в `/admin`

- [ ] **Тесты**
  - [ ] `pytest -q` проходит локально
  - [ ] `npm run build` проходит во `frontend/`

- [ ] **Бэкапы**
  - [ ] Проверено создание бэкапа: `./scripts/backup_db.sh`
  - [ ] Настроен cron/планировщик (пример в `docs/RUN.md`)

## После релиза

- [ ] Проверить `/health`
- [ ] Проверить логин во фронтенд и доступность `/admin`
- [ ] Проверить критические сценарии:
  - [ ] создание заказа
  - [ ] смена статуса заказа через workflow
  - [ ] добавление платежа + возврат
  - [ ] создание счета с эмитентом
  - [ ] склад: создание товара + корректировка остатка
